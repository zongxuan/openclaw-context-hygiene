# Context-Hygiene 插件详解

> 让 AI 对话长期保持快速、稳定的"清洁工"

---

## 📖 前言：什么是 Context？

在开始之前，我们需要理解一个概念：**Context（上下文）**。

当你和 AI 对话时，AI 需要"记住"你们聊过的内容。这个"记忆"就是 Context——它包含了：
- 你之前说过的话
- AI 之前回复过的话
- AI 调用过的工具（如读取文件、执行命令）
- 工具返回的结果

**问题来了**：对话越来越长，Context 就越来越大。当 Context 大到一定程度时：
- AI 回复变慢
- 可能超出 AI 的"记忆容量"（上下文窗口）
- AI 开始"遗忘"重要信息

**Context-Hygiene 就是来解决这个问题的**——它像一个尽职的清洁工，自动清理 Context 中的垃圾，让 AI 始终保持最佳状态。

---

## 🤔 什么是 Context-Hygiene？

**Context-Hygiene** 是一个 OpenClaw 插件（Hook），它的作用是**自动压缩和清理 AI 对话的上下文**，让对话在长时间运行后依然保持快速和准确。

### 生活中的类比 🧹

想象你在写一本很长的书：
- 每一次对话就像往书里加一页
- 写到第 100 页时，翻到前面找内容很慢
- **Context-Hygiene 就像一个编辑**：定期帮你把重复的内容合并、把没用的草稿删掉、把长段落缩短

---

## 🏗️ 工作原理：三层清洁机制

Context-Hygiene 使用**三层清洁机制**，就像一个有三道关卡的过滤器：

```
你发送消息
    │
    ▼
┌─────────────────────────────────────────┐
│  第一层：snipCompact（快速清洁）          │
│  耗时：~5毫秒                            │
│  • 删除"僵尸"消息                        │
│  • 删除重复的工具结果                     │
│  • 删除空消息                            │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  第二层：autoCompact（AI自动总结）         │
│  耗时：~10-30秒                          │
│  • AI 自动总结较早的对话                  │
│  （这是 OpenClaw 内置功能）              │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  第三层：contextCollapse（深度清洁）      │
│  耗时：~5毫秒                            │
│  • 压缩中间结果                          │
│  • 跨对话去重文件读取                     │
│  • 截断过长的输出                         │
└─────────────────────────────────────────┘
    │
    ▼
    AI 收到精简后的上下文，快速回应！
```

---

## 🔬 逐层解析

### 第一层：snipCompact（快速清洁）

这一层**最快**，每次 AI 生成回复前都会运行。它的任务是：

#### 1. 删除僵尸消息 🧟
- **什么是僵尸消息？** 被删除或失效的消息，标记为 `[zombie message]` 或 `[deleted message]`
- **为什么要删？** 这些消息已经没有内容了，只会浪费空间

#### 2. 删除重复的工具结果 🔄
- **场景举例**：AI 读取同一个文件两次，第二次的结果和第一次一样
- **怎么处理？** 保留最新的，删除旧的重复内容
- **怎么判断是重复的？** 用 SHA1 哈希比较内容——不是简单截取开头，而是真正比较内容是否相同

#### 3. 删除连续重复的文件读取 📄
- **场景举例**：连续读取同一个文件 5 次
- **怎么处理？** 只保留最后一次读取的结果

#### 4. 保护重要消息 🛡️
- **什么是重要消息？** `role: 'system'` 的消息，比如 SOUL.md（AI 的人设）、AGENTS.md（指令文件）
- **为什么保护？** 这些是 AI 的"灵魂"和"记忆"，绝对不能丢失

---

### 第二层：autoCompact（AI 自动总结）

这一层由 OpenClaw 自动触发，不需要配置。它会：
- 让 AI 把较早的对话压缩成简短的总结
- 类似于人类会做："上周我们讨论了 A、B、C三点"

---

### 第三层：contextCollapse（深度清洁）

这一层在" compaction"（压缩）前运行，做更激进的处理：

#### 1. 折叠中间结果 📦
- **场景**：AI 调用了一个工具（比如执行命令），返回了 1000 行结果。但后面的对话只需要知道"命令执行成功了"就够了
- **怎么处理？** 把 1000 行结果替换成一句话：`[Collapsed intermediate result for "path/to/file"]`

#### 2. 跨对话去重文件读取 🔍
- **场景**：在整个对话历史中，读取了同一个文件多次
- **怎么处理？** 只保留最后一次读取的结果，中间的都折叠

#### 3. 截断超长输出 ✂️
- **场景**：某个工具返回了 5000 字的内容，但 maxChars 设置为 1000
- **怎么处理？** 保留前 1000 字，加上一句 `[Truncated 5000→1000]`

---

## 📊 实际效果

假设你的对话有 100 条消息（原始大小约 50,000 tokens）：

| 处理阶段 | 消息数 | Tokens | 节省 |
|---------|-------|--------|------|
| 原始 | 100 | 50,000 | - |
| 第一层 snipCompact | ~85 | ~35,000 | 30% |
| 第二层 autoCompact | ~60 | ~20,000 | 43% |
| 第三层 contextCollapse | ~55 | ~15,000 | 25% |

**最终**：50,000 tokens → 15,000 tokens，AI 依然知道所有关键信息！

## ⚙️ 安装和配置

### 安装

```bash
# 方式1：从 npm 安装（待发布后）
npm install openclaw-context-hygiene

# 方式2：手动安装
# 把插件文件夹复制到 OpenClaw 的 hooks 目录
cp -r openclaw-context-hygiene ~/.openclaw/hooks/

# 重启 OpenClaw
openclaw gateway restart
```

### 配置

**无需配置**，开箱即用！

插件会使用默认配置：
- `truncateMaxChars: 2000`（每个工具结果最多保留 2000 字符）

如需自定义，可以直接编辑 `index.js` 文件。

## ❓ 常见问题

### Q: 会把我的 SOUL.md 内容删掉吗？
**不会**。从 v3.0.0 开始，所有 `role: 'system'` 的消息都受到保护，不会被压缩。

### Q: 怎么知道插件在工作？
插件会在处理后输出统计信息到 `_contextHygiene` 字段，包含：
- `zombiesRemoved`：删除了多少僵尸消息
- `dupeToolResultsRemoved`：删除了多少重复工具结果
- `charsSaved`：总共节省了多少字符

### Q: 可以关闭某些清洁功能吗？
可以。需要直接编辑 `index.js` 中的对应函数。

### Q: 会影响 AI 的回答质量吗？
**不会**。插件只会删除或压缩**无用的信息**（重复、空消息、中间结果），关键信息（你的问题、AI 的回答、系统指令）都会被保留。
